\section{Công Cụ Đảm Bảo Chất Lượng Mã Nguồn}

Project Dividex Admin Dashboard sử dụng một bộ công cụ hiện đại để đảm bảo chất lượng mã nguồn, kiểm thử tự động, và tuân thủ chuẩn lập trình.

\subsection{Framework Kiểm Thử}

\begin{itemize}
    \item \textbf{Vitest 4.1.2}: Framework kiểm thử chính, được xây dựng trên nền Vite, hỗ trợ kiểm thử nhanh và hot reload.
    \item \textbf{React Testing Library 16.3.2}: Thư viện kiểm thử component React, tập trung vào việc kiểm thử hành vi thay vì implementation detail.
    \item \textbf{@testing-library/jest-dom 6.9.1}: Cung cấp các matcher tùy chỉnh cho assertion trong kiểm thử DOM.
    \item \textbf{@vitest/coverage-v8 4.1.2}: Công cụ đo lường code coverage sử dụng V8 engine.
    \item \textbf{@vitest/ui 4.1.2}: Giao diện trực quan cho Vitest, cho phép xem kết quả kiểm thử trực tuyến.
\end{itemize}

\subsection{Công Cụ Kiểm Soát Chất Lượng Mã}

\begin{itemize}
    \item \textbf{ESLint 9.39.1}: Công cụ linting cho JavaScript/TypeScript, phát hiện lỗi và kiểu lập trình không tuân thủ chuẩn.
    \begin{itemize}
        \item Plugin: eslint-plugin-react-hooks 7.0.1 (kiểm tra quy tắc React Hooks)
        \item Plugin: eslint-plugin-react-refresh 0.4.24 (tối ưu Fast Refresh)
        \item Config: typescript-eslint 8.46.4 (hỗ trợ TypeScript)
    \end{itemize}
    \item \textbf{Prettier 3.8.0}: Công cụ định dạng mã tự động, đảm bảo tính nhất quán về style code.
    \item \textbf{TypeScript 5.9.3}: Ngôn ngữ lập trình type-safe cho JavaScript, giúp phát hiện lỗi tại thời điểm compile.
\end{itemize}

\subsection{Công Cụ Build và Development}

\begin{itemize}
    \item \textbf{Vite 7.2.4}: Build tool hiện đại với cold start nhanh và hot module replacement.
    \item \textbf{React 19.2.0}: Framework frontend chính.
    \item \textbf{React Router 7.12.0}: Quản lý routing cho ứng dụng single-page.
\end{itemize}

\section{Cấu Hình Môi Trường Kiểm Thử}

\subsection{Nền Tảng Phát Triển}

\begin{itemize}
    \item \textbf{Node.js}: Phiên bản 24.12.0 hoặc tương thích
    \item \textbf{Package Manager}: npm (hoặc yarn, pnpm - tương thích)
    \item \textbf{Node Modules}: Được cài đặt qua \texttt{npm install}
\end{itemize}

\subsection{Trình Duyệt Kiểm Thử}

\begin{itemize}
    \item \textbf{JSDOM}: Môi trường DOM mô phỏng cho Vitest (default)
    \item \textbf{Trình duyệt thực}: Chrome, Firefox (hỗ trợ test manual)
    \item \textbf{Headless Mode}: Vitest chạy ở chế độ headless (không cần giao diện trình duyệt)
\end{itemize}

\subsection{Cách Chạy Kiểm Thử}

\subsubsection{Chạy Kiểm Thử Cơ Bản}
\begin{verbatim}
npm test
\end{verbatim}
Chạy tất cả các test files trong chế độ watch (tự động chạy lại khi code thay đổi).

\subsubsection{Chạy Với Giao Diện Trực Quan}
\begin{verbatim}
npm run test:ui
\end{verbatim}
Mở giao diện web Vitest UI để xem kết quả kiểm thử trực quan, có thể theo dõi từng test case.

\subsubsection{Chạy Một Lần (CI/CD Mode)}
\begin{verbatim}
npm run test:run
\end{verbatim}
Chạy tất cả test một lần (không watch mode), thích hợp cho pipeline CI/CD.

\subsubsection{Đo Lường Code Coverage}
\begin{verbatim}
npm run test:coverage
\end{verbatim}
Chạy tất cả test với đo lường coverage, tạo ra report HTML trong thư mục \texttt{coverage/}.

\subsection{Cấu Hình Vitest}

Tệp \texttt{vitest.config.ts}:
\begin{itemize}
    \item \textbf{Environment}: \texttt{jsdom} (môi trường browser được mô phỏng)
    \item \textbf{Globals}: Kích hoạt (cho phép sử dụng \texttt{describe}, \texttt{it}, \texttt{expect} mà không cần import)
    \item \textbf{Setup Files}: \texttt{vitest.setup.ts}
    \item \textbf{File Pattern}: \texttt{src/**/*.{test,spec}.{ts,tsx}}
    \item \textbf{Coverage Provider}: v8
    \item \textbf{Coverage Reporters}: text, json, html
\end{itemize}

\subsection{Môi Trường Ứng Dụng}

\begin{itemize}
    \item \textbf{Development}: \texttt{npm run dev} hoặc \texttt{npm run dev:test} (dengan VITE\_USE\_MOCK=true)
    \item \textbf{Build}: \texttt{npm run build} (compile TypeScript và bundle với Vite)
    \item \textbf{Preview}: \texttt{npm run preview} (preview bản build)
\end{itemize}

\subsection{Best Practices Được Áp Dụng}

\begin{enumerate}

    \item \textbf{Mocking}: Dependencies được mock để cô lập code under test
    \item \textbf{Cleanup}: afterEach hooks xóa state để tránh test pollution
    \item \textbf{Descriptive Test Names}: Tên test rõ ràng mô tả behavior
    \item \textbf{Arrange-Act-Assert Pattern}: Test structure rõ ràng
    \item \textbf{React Testing Library Best Practices}: Test by user behavior, không implementation
\end{enumerate}
